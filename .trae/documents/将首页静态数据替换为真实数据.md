# 将首页静态数据替换为真实数据

## 1. 分析现状
- 首页组件使用静态数据进行展示
- 项目已配置完整的API请求机制
- 缺少首页统计相关的API定义

## 2. 实现步骤

### 2.1 创建首页统计API模块
- 创建 `src/api/manage/dashboard.js` 文件
- 定义首页所需的统计API接口

### 2.2 修改首页子组件

#### 2.2.1 修改 `home-user-task-stats.vue`
- 引入工单统计API
- 在组件挂载时调用API获取真实数据
- 替换静态的 `userTaskStats` 数据

#### 2.2.2 修改 `home-sku-sale-stats.vue`
- 引入销售统计API
- 在组件挂载时调用API获取真实数据
- 替换静态的 `orderCountNum` 和 `orderAmountNum` 数据

#### 2.2.3 修改其他统计组件
- 依次修改 `sku-sale-rank-chart.vue`
- 修改 `sku-sale-collect-chart.vue`
- 修改 `partner-node-collect-chart.vue`
- 修改 `abnormal-equipment-table.vue`

### 2.3 统一数据请求处理
- 为所有统计组件添加加载状态
- 统一错误处理
- 确保数据格式与现有组件兼容

## 3. API接口设计

根据现有组件的数据结构，设计以下API接口：

| 接口名称 | 接口路径 | 方法 | 功能描述 |
|---------|---------|------|---------|
| 获取工单统计 | `/manage/dashboard/taskStats` | GET | 获取工单总数、完成工单、进行工单、取消工单等统计数据 |
| 获取销售统计 | `/manage/dashboard/saleStats` | GET | 获取订单量、销售额等统计数据 |
| 获取SKU销售排名 | `/manage/dashboard/skuSaleRank` | GET | 获取SKU销售排名数据 |
| 获取SKU销售汇总 | `/manage/dashboard/skuSaleCollect` | GET | 获取SKU销售汇总图表数据 |
| 获取合作商节点汇总 | `/manage/dashboard/partnerNodeCollect` | GET | 获取合作商节点汇总图表数据 |
| 获取异常设备列表 | `/manage/dashboard/abnormalEquipment` | GET | 获取异常设备列表数据 |

## 4. 技术要点

- 使用Vue 3的Composition API
- 利用axios进行HTTP请求
- 统一处理加载状态和错误
- 确保数据格式与现有组件兼容
- 遵循项目现有的代码风格和命名规范

## 5. 预期效果

- 首页所有统计组件显示真实数据
- 数据定期更新（可添加刷新机制）
- 统一的加载状态和错误提示
- 与现有系统无缝集成
